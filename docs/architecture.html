1 系统架构总览

Hub 层（本仓库）只做展示，不持有任何密钥。各社区自己运行同步 workflow， 结果以 results.json 形式部署到各自的 gh-pages。浏览器并行 fetch 所有社区的结果，纯前端渲染仪表盘。

2 单社区 Workflow 全流程（3 个 Job）

GitHub Actions workflow 的核心编排逻辑。Job 1 拉取仓库列表并切片； Job 2 通过 matrix 策略并行同步各批次（突破 6 小时限制）； Job 3 汇总结果、诊断失败、合并历史数据、部署到 gh-pages。

4 mirror_repos.py 单仓库同步流程

核心同步脚本的内部流程：对每个仓库执行 "下载 → 创建目标 → 清除保护 → 推送" 四步， 失败时自动分类错误原因（大文件 / push protection / hook 拒绝 / 超时 等 9 种类型）。
